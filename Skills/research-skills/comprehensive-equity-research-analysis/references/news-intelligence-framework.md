# 最新動向、消息與重要動態判讀框架 (News Intelligence Framework)

將「最新動向 / 消息 / 重要動態（happenings）」轉成可投資決策使用的結構化證據。

## 1) 覆蓋範圍（預設近 30-90 天）

優先追蹤以下重要動態：
- 財報、法說與指引調整
- 監管與政策變動
- 重大產品/技術里程碑
- 併購、資本配置、融資與回購
- 供應鏈、客戶、定價與庫存異動
- ETF/資金流與大戶持倉變化（若適用）
- 競爭者重大發布、價格策略、客戶設計 win/loss
- 產業敘事轉折、買方關注焦點、 sell-side 主流 debate 變化

另外必做兩件事：
- 建立 `前瞻重要動態雷達`：檢查未來 14-30 天的高重要性 upcoming happenings。
- 建立 `重要動態深讀`：檢查過去 14-30 天內剛發生的重要 recent happenings，做內容與市場解讀拆解。

若屬事件驅動型、新聞流驅動型或敘事驅動型公司/產業（例如半導體、AI、FDA、監管敏感產業），不可只做一般新聞摘要。

## 1.1) 前瞻重要動態雷達（必做）

至少回答：
- 未來 14-30 天有哪些高重要性 happenings？
- 市場目前在期待什麼？
- 市場主要爭論是什麼？
- 哪些內容可能已反映在股價？
- 什麼結果才算真正超預期 / 低於預期？

建議輸出格式：

| 日期 | 重要動態 | 類型 | 市場預期 | 關鍵爭點 | 若超預期 | 若不及預期 |
|---|---|---|---|---|---|---|
| YYYY-MM-DD |  | event / news / supply-chain / regulation / narrative |  |  |  |  |

## 1.2) 重要動態深讀（若過去 14-30 天內剛發生，必做）

至少回答：
- 實際發生了什麼？
- 哪些原本市場期待的內容沒有出現？
- 市場當天或隔日如何解讀？
- 你是否同意市場解讀？若不同意，錯在哪裡？
- 對營收、毛利、CapEx、競爭格局、估值中樞的傳導路徑是什麼？
- 這件事主要改變 Bull / Base / Bear 的哪個情境？

禁止只寫「某大會舉行」「某產品發布」「某媒體有報導」。若缺少對內容拆解、重要性判斷與市場解讀，視為未完成。

## 2) 確定性等級（必填）

- `A-Confirmed`：來自法定申報、公司公告、官方文件，可直接驗證。
- `B-High`：多個高可信來源一致，但非原始法定文件。
- `C-Medium`：單一可信來源或訊號不完整，需追蹤。
- `D-Low`：傳聞/二手轉述，未被可靠來源交叉驗證。

規則：
- 僅 `A`/`B` 可作為主要投資論點核心依據。
- `C`/`D` 只能列入觀察清單與情境風險，不可主導估值中樞。
- 若情報僅來自單一網路來源（單一媒體、論壇、社群貼文、轉述摘要），預設降級至少一級；通常歸類為 `C` 或 `D`。

## 3) 消息評價欄位（每則消息都要有）

| 日期 | 事件 | 來源代號 | 確定性 | 市場反應 | 你的評價 | 對營收/利潤/估值影響 | 納入核心論點? |
|---|---|---|---|---|---|---|---|
| YYYY-MM-DD |  | A1 | A/B/C/D | 正向/中性/負向 | 高度利多/中性偏空... | 短中長期傳導路徑 | 是/否 |

來源 URL：
- A1: `https://...`

## 4) 你的評價（必須明確）

至少回答：
- 這則消息是否改變基本面趨勢，或僅短期情緒？
- 對 `Bull/Base/Bear` 哪個情境影響最大？
- 若消息失真或延遲落地，哪個假設會被推翻？
- 若屬重大 happening，市場的主流解讀是否合理？你同意或不同意什麼？
- 若屬即將發生的重要動態，市場目前最可能錯估的是什麼？
- 這件事若不是正式 event，而是 news flow / customer move / supply-chain signal，為什麼它仍然重要？

## 5) 輸出要求

- 先主動搜尋 `important happenings`，再決定哪些應列入核心論點；不要等使用者點名。
- 若未來 14 天內存在高重要性 happenings，先寫 `重要動態雷達`，再寫一般消息表。
- 若過去 14-30 天內剛發生高重要性 happenings，必須加入 `重要動態深讀` 專節。
- 消息段落先列「已確認重點」，再列「待驗證訊號」。
- 表格內僅使用來源代號；完整來源 URL 放在表格下方區塊，提升可讀性。
- 若來源是網頁或社群貼文（含 X.com），必須提供可點擊的原始 URL（不可只寫媒體名稱或轉述）。
- 每項關鍵結論後附絕對日期。
- 對低確定性消息加註：`需二次驗證，暫不作為核心估值依據`。
