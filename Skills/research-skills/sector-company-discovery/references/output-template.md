# Output Template

## Search Scope

- sector / industry / theme
- geography
- market-cap or business-model filters
- analysis date
- what the user wants optimized for

## Candidate Set

- list the companies included
- state why each belongs
- state which obvious names were excluded and why if relevant

## What Matters In This Industry

- define the 3-6 dimensions that matter most
- explain why these dimensions matter

## Recent Topics And Narratives

- list the 1-3 most important recent topics affecting the sector
- explain the market narrative around each topic
- separate what happened from what the market believes happened
- say whether the narrative is supported by primary sources or is still speculative

## Comparison Table

Use a compact table with dimensions such as:

- business quality
- growth durability
- balance sheet
- valuation
- resilience
- capital allocation
- special risks

## Top Tier And Why

- identify the top 2-3 names
- explain the edge of each one

## Winner Or Preferred Shortlist

- choose one winner if justified
- otherwise give a top tier and explain what additional work decides the winner

## Main Reasons The Leading Idea Could Be Wrong

- state the most important counterpoints

## Next Step For Deep Research

- say whether to use `company-valuation-analyst`
- say whether to use `comprehensive-equity-research-analysis`
- list the specific open questions to solve next

## Delivery Rules

- Write the report in Traditional Chinese.
- Save it as a Markdown file in `Reports/<YYYY-MM-DD>/`, where `<YYYY-MM-DD>` is the analysis date.
- Use the filename `sector-company-discovery-{YYYY-MM-DD}-00.md`, or the next available suffix if that date already exists.
- After saving the report, stage the report file and push the commit.
