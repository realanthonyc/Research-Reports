# Microsoft 估值研究備忘錄

分析日期：2026-03-19  
標的：Microsoft (`MSFT`)  
市場快照：截至 2026-03-11，股價約 `404.88` 美元，市值約 `3.01T` 美元，企業價值約 `3.04T`，TTM `P/E` 約 `25.34x`、Forward `P/E` 約 `23.07x`。[StockAnalysis](https://stockanalysis.com/stocks/msft/)

## Snapshot

Microsoft 目前是最典型的 AI 平台型 mega-cap 之一，不是純 AI 標的，而是把 AI 做成企業產品、雲端基礎設施與開發者工具的分發平台。市場現在看的不是「它能不能做 AI」，而是「Azure、Copilot、OpenAI / Azure 生態、以及大規模 AI 基建投資，最後能不能轉成可持續每股價值」。

最新官方財報顯示，FY26 Q2 營收 `813 億美元`、年增 `17%`，`Azure and other cloud services` 年增 `39%`，commercial RPO 到 `6,250 億美元`，其中約 `45%` 與 OpenAI 相關；同季 CapEx 為 `375 億美元`，其中約三分之二是 GPU / CPU 等短壽命資產。[Microsoft FY26 Q2](https://www.microsoft.com/en-us/investor/earnings/fy-2026-q2/press-release-webcast) [Microsoft metrics](https://www.microsoft.com/en-us/investor/earnings/fy-2026-q2/metrics)

## Business model in plain language

Microsoft 的本質是企業軟體 + 雲平台 + 生產力工具 + 開發者平台的複合型現金機器。

AI 對它的意義不是單一產品，而是四層：

1. `Azure` 提供 AI 訓練與推理算力。
2. `Microsoft 365 / Copilot` 把 AI 變成訂閱升級。
3. `GitHub / Developer tools` 把 AI 納入開發流程。
4. `Security / Dynamics / Data platform` 讓 AI 變成企業工作流的一部分。

因此，Microsoft 的核心問題不是需求，而是 `monetization efficiency`：AI 投資很大，但能否以足夠快的速度轉成高質量收入與自由現金流。

## What drives value

最重要的價值驅動因子有四個：

1. `Azure` 持續增長與份額擴張。
2. `Copilot` 與 AI 功能的付費轉化率。
3. `commercial RPO` 的轉化與可見度。
4. AI 基建 CapEx 是否在 2026 下半年開始正常化，否則 FCF 會被短期壓住。

從最近財報看，最值得注意的是：

- Azure 與其他雲服務已經是明確的雙位數強成長。
- commercial RPO 已經很大，代表未來收入可見度高。
- 但 CapEx 也非常大，而且偏短壽命資產，意味著近中期自由現金流會被壓縮。

## Balance sheet and dilution

Microsoft 的資產負債表仍然非常強。

最新 balance sheet 顯示：

- 現金與短期投資：`894.62 億美元`
- 長短期債務合計：約 `402.62 億美元`
- 淨現金：大約 `492 億美元`
- 流通股：`74.29 億股`

也就是說，Microsoft 不是靠融資撐 AI 的公司，而是靠自有現金流與資產負債表去撐。這一點跟 Oracle 這類重槓桿 AI 基建股很不同。[Microsoft balance sheet](https://www.microsoft.com/en-us/investor/earnings/fy-2026-q2/balance-sheets)

因此，`dilution risk` 對 Microsoft 不是主要風險。真正的風險是 `capex efficiency` 與 `multiple compression`。

## Scenario valuation table

Microsoft 比較適合用 `forward earnings multiple` + `cash flow durability` 來看，而不是用事件驅動式 rNPV。

| Scenario | Assumptions | Implied fair value | 代表含義 |
| --- | --- | --- | --- |
| Bear | Azure 成長降到中高 teens，AI CapEx 繼續高位，市場願意給的 multiple 壓到約 `20x` | `360-380` 美元 | 近中期 FCF 受壓，市場開始重新定義 AI 回報節奏 |
| Base | Azure / Cloud 仍維持強勁，Copilot 與 RPO 逐步轉化，multiple 約 `23-24x` | `430-460` 美元 | AI 是真增長，但不會完全免費，屬於合理溢價 |
| Bull | AI diffusions 速度優於預期，Azure monetization 更強，CapEx 開始明顯正常化，multiple `26-28x` | `500-540` 美元 | Microsoft 變成「AI 平台 + 企業工作流」的長期核心複合成長股 |

我的基準判斷是：

- 現價約 `404.88` 美元。
- Base case fair value 約落在 `440` 美元附近。
- 也就是說，Microsoft 不是明顯便宜，但也不是已經把所有好消息都吃光。

## What the market is pricing in now

市場現在大致在 price in 下面幾件事：

- Azure 仍能維持高雙位數到接近 40% 的強勁成長。
- OpenAI / Copilot / Azure 的 AI 需求可持續擴大。
- `625B` 的 commercial RPO 會逐步轉成收入。
- AI 基建 CapEx 雖大，但最後能被規模經濟與軟體毛利消化。

這代表 Microsoft 已經不是「便宜的 AI 下注」，而是「高質量平台 compounder 的合理溢價」。如果把它當成 deep value，會錯；如果把它當成零風險 AI 資產，也會錯。

## Recent topics and narratives

最近 30-90 天，Microsoft 的敘事主線是：

1. `AI capex 先行，收入後到`：Q2 CapEx `375 億美元`，但 management 明確表示需求仍大於供應。這支持了 AI 需求真實存在，但也說明短期 FCF 不會漂亮。[Microsoft earnings call](https://www.microsoft.com/en-us/investor/events/fy-2026/earnings-fy-2026-q2)
2. `Azure 是真正的 monetization engine`：Azure and other cloud services 年增 `39%`，這不是概念，而是已經在財報裡看到的現金化入口。[Microsoft FY26 Q2](https://www.microsoft.com/en-us/investor/earnings/fy-2026-q2/press-release-webcast)
3. `RPO 規模極大，但 OpenAI concentration 也高`：商業 RPO 到 `625B` 很驚人，但其中約 `45%` 來自 OpenAI，這既是強勢訊號，也是集中度風險。[Microsoft earnings call](https://www.microsoft.com/en-us/investor/events/fy-2026/earnings-fy-2026-q2)
4. `市場辯論點不是 AI 能不能賺錢，而是賺錢速度與估值倍數`：目前的爭論焦點是 capex 回報與毛利壓力，而不是 Microsoft 是否能參與 AI。

## Verdict

如果你問我 Microsoft 現在值不值得持有，我的結論是：

- `持有` 沒問題，因為它的質量、現金流與 AI 平台位置都很強。
- `追價` 不吸引，因為它已經是 mega-cap 高品質定價，不是便宜貨。
- `逢回分批加碼` 比 `直接追高` 更合理。

一句話版本：

Microsoft 是 `高質量 AI 平台 compounder`，不是 `deep undervalued`；它的風險不在商業模式，而在 `AI capex 回報` 與 `估值倍數是否再擴張`。

## Critical unknowns

- AI capex 何時從高壓投入轉向較正常的折舊與維護節奏。
- Copilot 與 Azure AI 的付費轉化是否足以支撐更高的每股 FCF。
- OpenAI 相關營收與合作架構的長期經濟分配。
- 若 memory、GPU、網路與 datacenter 供應再度吃緊，Microsoft 的供需缺口會拖多久。

## What to watch next

- 下次財報中的 `Azure` 成長率與 `CapEx` 走勢。
- `commercial RPO` 的轉化速度。
- `free cash flow` 是否開始明顯修復。
- Copilot / AI 功能的 ARPU 提升是否能持續。
