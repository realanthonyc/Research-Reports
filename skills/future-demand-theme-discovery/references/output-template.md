# Output Template

Use this order unless the user explicitly asks for another format.

## 1. 研究目標與時間框架

- 說明本次要找的是未來 `6-24 個月` 需求可能明顯加速、且可持續 `1-2 年以上` 的主題
- 說明地區、資產範圍、是否偏向美股或全球
- 說明本次要找的是「現在還沒完全被吹上風口，但很有機會成為下一個熱門焦點」的主題，而不是今天已經最擁擠的交易

## 2. 候選主題池與篩選漏斗

先列出本次研究過、掃過、或納入比較的候選主題，至少分成：

- `最終入選`
- `有吸引力但暫不入選`
- `研究後排除`

對每個未入選主題說明至少一個明確原因，例如：

- 需求時間點太早
- 可持續性不足
- 供給反應太快
- 已過度擁擠
- 價值捕捉位置不清楚
- 目前較像景氣循環而非結構需求
- 已經太過主流，雖然方向正確但不夠早

如果有看到 `聰明錢 / 專業資金 / 產業鏈提前布局`，可在這一節列出作為 hint，但要明確標示：

- 這只是加分訊號
- 不是主題成立的單獨證據

## 3. 結論先行：Top 3-5 未來需求主題

對每個主題列出：

- `主題名稱`
- `一句話結論`
- `目前階段`：過早 / 接近拐點 / 正在放量 / 已偏擁擠
- `優先級`：高 / 中 / 觀察
- `為何還不算 fully crowded`

## 4. 入選主題完整分析

這一節改成 `每個主題一個完整 section`。
不要用「先討論所有主題的勝出原因，再討論所有主題的需求加速原因」這種橫向切法。
必須完成一個主題的完整分析後，再進入下一個主題。

每個主題都使用以下子結構：

### 主題 X：{主題名稱}

**為何能從候選池中勝出**

- 為何它比其他候選更符合 `6-24 個月需求加速`
- 為何它比其他候選更有 `1-2 年以上` 的持續性
- 為何它的價值捕捉位置更清楚
- 為何它的需求規模可能足夠大
- 為何它目前仍值得深入研究與等待埋伏

**為何需求會在未來加速**

- 結構性驅動因子
- 為何時間點落在未來幾季到一年
- 為何不是一次性事件
- 為何需求可能大到足以影響產業 earnings / 市場關注

**為何可持續 1-2 年以上**

可用證據包括：

- 多年預算
- backlog / bookings
- 法規或合規需求
- 長建設週期
- 生態系承諾

**產業鏈與價值捕捉位置**

- 上游 / 中游 / 下游 / 基礎設施
- 真正可能高回報的環節
- 哪些只是敘事受益者，哪些才可能是股東受益者

**值得先研究的代表公司或受益籃子**

- `核心受益者`
- `次級受益者`
- `需要避免的純概念股`

如果暫時不適合點名個股，也要說明原因。

**市場是否尚未完全定價**

- 市場是否仍低估需求
- 是否把主題誤當 cyclical / niche / overhyped
- 哪些環節仍可能被忽視
- 估值與敘事是否已經過熱

**驗證訊號與 thesis breakers**

- `接下來 1-2 季要追蹤的驗證訊號`
- `使 thesis 失效的因素`

**埋伏策略與等待條件**

- 什麼類型的回調值得關注
- 哪些短期雜音可能帶來錯殺
- 哪些跡象代表還不該埋伏

## 5. 跨主題比較與排序理由

- 為何這幾個主題的優先級不同
- 哪些最適合立刻深挖
- 哪些更適合放進觀察名單等待驗證

## 6. 主要風險與不確定性

- 總結本次研究最大的幾個不確定點

## 7. 資料來源

- 優先列出主要來源
- 區分官方資料、財報、政策文件、學術研究、可靠媒體
